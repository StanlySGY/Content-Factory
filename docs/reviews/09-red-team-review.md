## 2. 审查目标

以攻击者视角审查设计的安全边界、权限、数据保护、外部接入与审计完整性，发现可被滥用的薄弱点。

## 3. 审查清单

- [ ] 权限提升路径：Agent / MCP / 插件能否突破授权边界
- [ ] 敏感数据泄露：上下文包、日志、消息是否可能外泄敏感信息
- [ ] 第三方 MCP 供应链风险：来源校验、沙箱、权限最小化
- [ ] Agent 越权：能否绕过 Gateway / ToolRouter 执行未授权动作
- [ ] 发布越权：能否绕过审核或授权直接发布
- [ ] 审计绕过：关键操作是否存在不可审计路径
- [ ] 密钥与凭证：是否存在明文存储或过度暴露
- [ ] 注入风险：Prompt 注入、工具输入、外部内容注入
- [ ] 破坏性操作：是否需强确认且可恢复
- [ ] 多租户 / 项目边界（后续阶段）隔离假设是否成立
- [ ] 失败与降级是否被滥用为绕过控制的手段
