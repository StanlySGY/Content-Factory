## 2. 审查目标

验证 MCP 生命周期、注册、安装、启停、权限、日志、热加载、市场与第三方治理是否完整、安全、可扩展。

## 3. 审查清单

- [ ] 网关隔离成立，业务 / Agent / Skill / 插件不直连 MCP Server
- [ ] 生命周期状态机完整且自洽
- [ ] 注册、安装、启停、热加载、卸载流程可执行
- [ ] 权限模型分层叠加取最小授权，维度完整
- [ ] 风险分级（low / medium / high）与默认策略清晰
- [ ] 日志覆盖生命周期、调用、权限、错误、健康，敏感数据脱敏
- [ ] 热加载范围与回滚策略明确
- [ ] 市场与第三方 MCP 治理（Manifest、来源、校验、风险）完整
- [ ] 新增第三方 MCP 无需修改业务代码
- [ ] MCP Server 不承载核心业务规则
- [ ] 密钥仅以引用注入，无明文存储
- [ ] 数据模型映射明确，缺口已标注
