## 2. 审查目标

验证页面树、布局、组件、交互与各核心页面设计是否完整、一致、可用、合规。

- [ ] 长任务实时更新通道（WS/SSE）已定义（缺失，UI-001 Critical）
- [ ] 核心模块齐全：工作流设计器 / Skill / 插件 / 身份角色（缺口 UI-002~004）

## 5. 修复建议

- **UI-001（Critical）**：新增实时通道章节（推荐 SSE 流式输出 + 状态推送），定义订阅粒度（session/stage_run/task）、断线重连与回退轮询、消息类型（status_change/agent_token/tool_call/error）、与"前端状态非权威"原则的关系。
- **UI-002~006**：补"工作流设计器"页面（阶段/依赖/执行者/门禁/模板版本化）；补 Skill/插件管理界面；明确登录/会话、项目成员与角色、权限页结构、项目切换与隔离表达；补渠道配置 + 发布记录版本锚定 + 审计可见性 + 插件化渠道抽象；新增 UI→架构 §3.1 模块映射表，明确执行监控/审查工作台承载页。
- **UI-007**：补全各页空态 + 全局错误态（鉴权/网络/超时/实时断连）+ 加载/分页失败处理。
- **UI-008~011**：统一信息架构图与页面树；明确调用追溯视图；补状态徽章映射表；声明高风险确认清单由后端风险策略驱动、前端仅渲染。

## 6. 最终结论

有条件通过 —— 视觉/交互原则与页面骨架扎实，但实时更新通道缺失（UI-001，Critical）与工作流设计器/Skill/插件/身份角色/发布渠道等核心模块界面缺口（UI-002~006）必须补齐后方可进入开发。
